# Boundary Coach AI — 測試案例

## 一、自由輸入影響分析

### 測試輸入

| 欄位 | 內容 |
|------|------|
| 關係 | 父母 |
| 對方 | 媽媽（稱謂：媽） |
| 對方訊息 | 你現在都不回家，養你這麼大有什麼用？ |
| 自由輸入 | 我其實前兩週都有回家，只是這週工作很累。我不是不在乎家人，只是我真的需要休息。我希望媽媽知道我有努力，但不要用責備的方式跟我說話。 |

### 操作步驟

1. 首頁填入上表內容（「想補充的背景或故事」欄位）
2. 完成旅程階段 1–7（可快速點選選項）
3. 進入第 8 階段「陪伴整理」，等待 AI 回傳

### 預期結果

1. **我聽見你說的是**：必須提到「前兩週都有回家」
2. **我聽見你說的是**：必須提到「這週工作很累」
3. **你可能真正想替自己說的是**：必須提到「不是不在乎家人」
4. **回覆建議**：至少兩個版本（溫和／理性／界線／極簡）須含「前兩週有回家」或「這週需要休息」
5. **不可**只產生「我最近比較忙，希望你理解」等通用句

### 程式驗證

回傳 JSON 後可於開發者工具執行（需先取得 `comprehensiveInsights` 物件）：

```ts
import { validateFreeInputTestCase } from "@/lib/validation/insightsExpectations";

validateFreeInputTestCase(comprehensiveInsights);
// checks 陣列應全部 passed: true
```

## 二、安定背景聲

### 測試流程

1. 進入第 2 階段「你的身體還好嗎？」
2. 身體感受選項下方出現背景聲選擇區塊
3. 預設為「不需要聲音」，**不會**自動播放
4. 選擇「雨聲」→ 點「讓聲音陪我一下」→ `rain.mp3` 播放
5. **播放中直接點「海浪聲」** → 應可切換，`ocean.mp3` 開始播放（有 fade）
6. **點「慢慢繼續」換到下一頁** → 音樂應持續播放，畫面下方出現迷你控制列
7. 點「先暫停一下」→ 聲音停止
8. 調整音量滑桿
9. 選「不需要聲音」或迷你列「關閉」→ 全部停止
10. 回到首頁／重新開始 → 音樂停止
11. 若 `public/audio/` 無音檔 → 顯示「目前無法播放背景聲，你仍然可以先做三次深呼吸。」

### 手機版檢查

- Safari / Chrome 開發者工具切換手機尺寸
- 確認選項按鈕可點、音量滑桿可拖、播放／暫停／關閉可用
- iOS 需使用者手動點擊才會播放（符合規格，無自動播放）

### 更換音檔

將 MP3 放入 `public/audio/`：

- `rain.mp3`
- `ocean.mp3`
- `spa-bamboo.mp3`
- `soft-piano.mp3`

詳見 `public/audio/README.md`。
